# Voice Identity v0.1.0 Release Notes (Draft)

## Overview

Voice Identity v0.1.0 is the first release-oriented package for the standalone Home Assistant custom integration. This release focuses on architecture-aligned service surfaces, privacy-safe projections, production-hardening validations, and HACS delivery readiness.

## Major Capabilities

- Diagnostics provider service (`voice_identity.get_diagnostics`)
- Repair recommendation service (`voice_identity.get_repairs`)
- Health service (`voice_identity.get_health`)
- Telemetry service (`voice_identity.get_telemetry`)
- Advisory attribution service (`voice_identity.attribute_speaker`)
- Identity context projection service (`voice_identity.get_identity_context`)
- Compatibility and migration validation matrix
- Performance and resource hardening baselines
- Fault injection and recovery hardening baselines
- Release-readiness and operational runbook validation

## Privacy Guarantees

Voice Identity public outputs are privacy-safe projections and do not expose:

- raw audio
- enrollment recordings
- transcripts
- vectors or embeddings
- fingerprint payload internals
- storage paths
- provider internals beyond contract-safe metadata
- registry internals outside public contracts
- secrets or tokens
- exception traces in public payloads

## Operational Status

Production-hardening artifacts VI-121 through VI-129 are implemented and validated by repository test suites.

## HACS Installation Note

This release is intended for installation through HACS as a custom repository (Integration type). It does not claim listing in the default HACS store.

## Known Limitations

- Attribution is advisory evidence and not identity truth.
- Identity Context is behavioral context and not authentication or authorization.
- No external telemetry export is included.
- Release workflows must pass in GitHub Actions before tagging and publishing.
